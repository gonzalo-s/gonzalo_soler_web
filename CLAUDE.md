# Project

Personal digital resume / portfolio site. A statically generated Next.js app
that showcases projects, skills, and experience. Content is sourced from Google
Sheets (used as a headless CMS) and baked into the build.
(Reference: see README.md for full details.)

# Stack

- Runtime: Node 20 (`@types/node@^20`)
- Framework: Next.js 15.2.8 (App Router), React 19
- Language: TypeScript 5
- Styling: SCSS / Sass (with Stylelint)
- Package manager: npm (see `package-lock.json`)
- Key dependencies:
  - `next@15.2.8` — framework, SSG, App Router
  - `react@^19` / `react-dom@^19` — UI
  - `papaparse@^5.5.3` — parses Google Sheets CSV exports
  - `react-icons@^5.5.0` — icon set
  - `@vercel/analytics` + `@vercel/speed-insights` — analytics

# Architecture

Brief summary only — see README.md for the full picture (esp. the
"Google Sheets as a headless CMS" section).

- Folder structure (`src/`):
  - `app/` — Next.js App Router pages, layouts, and routes
  - `components/` — reusable React components (colocated styles)
  - `constants/` — static constants and inline SVG icon definitions
  - `contexts/` — React context providers
  - `hooks/` — custom React hooks
  - `lib/services/` — content layer: CSV fetching, parsers, and config
  - `styles/` — global SCSS
  - `types/` — shared TypeScript types (e.g. `sections.ts`)
- Key conventions:
  - Content is fetched from Google Sheets CSV exports at build time (SSG);
    URLs live in `src/lib/services/config/csvUrls.ts`.
  - Each content section has a dedicated parser in
    `src/lib/services/parsers/` that transforms raw CSV rows into typed objects.
  - All CSV rows and parsed shapes are strictly typed
    (`src/lib/services/types/csvTypes.ts`, `src/types/sections.ts`).
  - Path alias: `@/*` maps to `./src/*`.
  - No runtime dependency on Google Sheets — content is baked into the build.
- Files Claude must never modify:
  - `package-lock.json` (managed by npm)
  - `.next/`, `out/`, `build/` (generated build artifacts)
  - `next-env.d.ts` (generated by Next.js)
  - any `.env` / `.env.*` files (secrets; only `.env.example` is tracked)

# Code style

- Use ES modules (import/export), not CommonJS (require).
- Max file length: 300 lines.
- Max function length: 150 lines.
- TypeScript strict mode is on (`tsconfig.json`) — keep code type-safe; avoid `any`.
- Prefer named exports; keep components small and colocate component styles.
- Follow Prettier formatting (`.prettierrc`): single quotes, semicolons,
  100+ char print width, trailing commas. Run `npm run format` to apply.
- Lint SCSS with Stylelint (`.stylelintrc.json`).

# Workflow

- Always run `npm run typecheck` after a series of changes.
- Always run `npm run lint` and `npm run format` before committing.
- Work in a feature branch, never directly on `main`.
- Commit messages: conventional commits format (feat:, fix:, chore:, etc.).
